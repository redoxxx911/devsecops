Final DevSecOps Report

1. Project Presentation

This project presents a complete DevSecOps delivery chain for a Zero-Trust Vault application. The application is composed of a FastAPI backend, a React frontend, an ELK monitoring stack, and a SecOps automation bot connected to GitLab, Elasticsearch, Telegram, and an LLM provider.

The objective is to demonstrate how security can be integrated into each phase of software delivery: code validation, static analysis, container build, image scanning, dynamic application testing, deployment, monitoring, and automated incident assistance.

3. CI/CD Pipeline Explanation

The global GitLab pipeline contains nine stages: install, lint, test, SAST, build, container scan, push, DAST, and deploy.

3.1 Install

The install stage prepares Python dependencies for the backend and validates that dependency files are usable inside CI.

3.2 Lint

The lint stage checks backend code with flake8 and frontend code with npm run lint.

3.3 Test

The test stage runs backend tests with pytest and exports Cobertura coverage.

3.4 SAST

The SAST stage runs SonarScanner against backend, frontend, and SecOps bot source code.

3.5 Build

The build stage creates Docker images for the backend, frontend, and SecOps bot.

3.6 Container Scan

The container scan stage uses Trivy to inspect the generated Docker image artifacts for high and critical vulnerabilities.

3.7 Push

The push stage uploads images to GitLab Container Registry when the pipeline runs on the main branch.

3.8 DAST

The DAST stage runs OWASP ZAP baseline scanning against the deployed frontend.

3.9 Deploy

The deploy stage documents Docker Compose and Kubernetes deployment options.

5. Security Scan Result Analysis

5.1 SAST Findings

The backend contains a hardcoded JWT secret, permissive CORS, and demo credentials. These are acceptable for a classroom demonstration but must be remediated before production deployment.

The frontend contains a hardcoded API URL. This should be injected through environment-specific configuration.

5.2 Container Scan Findings

Trivy is configured to report high and critical vulnerabilities. The remediation process is to update base images, update dependencies, rebuild, rescan, and document accepted risks.

5.3 DAST Findings

OWASP ZAP checks the running frontend for missing headers, passive scan alerts, exposed endpoints, and weak browser security configuration.

5.4 Monitoring Findings

Filebeat sends container logs to Elasticsearch. The SecOps bot watches for repeated 401 and error events and can summarize incidents through Telegram.

8. Conclusion

This project demonstrates a complete DevSecOps workflow across the full software delivery lifecycle. The CI/CD pipeline validates the code, performs static analysis, builds Docker images, scans containers, runs dynamic security testing, and prepares deployment. The monitoring stack and SecOps bot extend the project beyond deployment by adding runtime visibility and automated incident assistance.

The main improvement areas are secret management, stricter CORS configuration, stronger authentication, environment-based frontend configuration, and enforcing pipeline gates once the project moves from demonstration mode to production mode.
